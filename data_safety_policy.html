Data Safety Policy

1. Introduction

Ensuring the safety of user data is a top priority for Medixbot. Our platform, designed for telemedicine and health data management, implements industry-standard security measures to protect personal, medical, and AI-generated health insights. This Data Safety Policy outlines how we collect, store, process, and protect user information.

2. Data Collection and Processing

2.1 Personal and Medical Data

  • User identity information (name, contact details, date of birth)
  • Medical records, prescriptions, and laboratory results
  • AI-analyzed health metrics, such as glucose levels (if using a CGM device)
  • Telemedicine consultation records (audio, video, and text-based)

2.2 Usage and Technical Data

  • Device metadata (device type, OS version, app version)
  • Log files and system performance data
  • Encrypted chat logs for AI assistant interactions

2.3 Data Anonymization and Encryption

  • Sensitive health data is anonymized where possible to ensure privacy
  • All stored data is encrypted using AES-256 encryption standards
  • Data transmission is secured using TLS 1.2 and higher

3. Data Storage and Retention

  • Personal and medical data is securely stored in HIPAA, GDPR, and SOC 2-compliant cloud infrastructure
  • User data is retained only as long as required by healthcare regulations and operational needs
  • After the required retention period, data is permanently deleted or anonymized

4. Access Control and User Rights

  • Only authorized personnel, such as healthcare professionals and compliance officers, have access to sensitive data
  • Users have the right to access, modify, or delete their personal data upon request
  • Multi-factor authentication (MFA) is enforced for platform access to prevent unauthorized usage

5. Data Sharing and Third-Party Services

Medixbot does not sell user data. Data sharing is limited to:

  • Healthcare professionals for medical consultation and prescriptions
  • Laboratories for diagnostic tests (with user consent)
  • Secure AI analytics services (for improving health insights, using anonymized data)
  • Regulatory authorities where legally required

6. Security Measures

  • End-to-End Encryption: Ensures secure telemedicine consultations
  • Role-Based Access Control (RBAC): Limits access to authorized users only
  • Regular Security Audits: Routine testing to identify and mitigate vulnerabilities
  • Automatic Session Expiry: Prevents unauthorized access after inactivity

7. Incident Response and Data Breach Protocol

  • In case of a data breach, affected users will be notified within legally mandated timeframes
  • Immediate action will be taken to contain and resolve security threats
  • Logs and forensic analysis will be performed to prevent future incidents

8. Compliance with Data Protection Laws

Medixbot complies with international data protection regulations, including:

  • General Data Protection Regulation (GDPR) (EU)
  • Health Insurance Portability and Accountability Act (HIPAA) (USA)
  • Personal Data Protection Law (KVKK) (Turkey)
  • Other applicable regional healthcare and privacy laws

9. User Responsibilities

  • Use strong passwords and enable multi-factor authentication
  • Keep their app updated to the latest version for security enhancements
  • Report any suspicious activity related to their accounts immediately
